# 网络账号运营诊断专家

当用户提到需要诊断、分析、优化网络账号运营时，使用此技能。

## 触发条件

- 用户提到"诊断账号"、"分析运营"、"账号优化"
- 用户提供品牌/门店名，需要分析其线上账号
- 用户提供账号链接，需要分析该账号数据
- 用户明确要求使用"网络账号运营诊断专家"技能

## 执行流程

### 步骤1：解析输入

获取用户输入：
- 如果是品牌/门店名 → 进入步骤2（搜索账号）
- 如果是账号链接 → 直接进入步骤3（数据抓取）
- 如果用户指定了平台 → 只诊断指定平台

### 步骤2：搜索账号（多层容错）

**尝试1：WebSearch 搜索**

```bash
# 搜索各平台账号
WebSearch("{{输入}} 小红书 账号")
WebSearch("{{输入}} 抖音 账号")  
WebSearch("{{输入}} 视频号 账号")
WebSearch("{{输入}} 公众号 账号")
```

从搜索结果中提取：
- 账号名称
- 账号链接
- 粉丝数（如果有）

**尝试2：如果WebSearch失败，使用WebFetch直接访问**

```python
# 尝试访问平台搜索页面
WebFetch(url="https://www.xiaohongshu.com/search_result?keyword={{输入}}", 
         prompt="提取搜索结果中的账号名称和链接")
```

**容错处理**：
- 如果所有平台都未找到 → 告知用户并询问是否手动提供账号链接
- 如果部分平台找到 → 只分析找到的平台

### 步骤3：数据抓取（多层容错）

对每个找到的账号，尝试以下方法抓取数据：

**方法A：playwright（如果可用）**

```bash
# 注意：安装的包是 @playwright/cli，命令是 `playwright` 不是 `playwright-cli`
# 路径：/c/Users/yu_sh/AppData/Roaming/npm/playwright.cmd

/c/Users/yu_sh/AppData/Roaming/npm/playwright.cmd open {{账号链接}}
sleep 3
/c/Users/yu_sh/AppData/Roaming/npm/playwright.cmd snapshot
# 读取快照文件
cat .playwright-cli/page-*.yml | tail -100
```

**方法B：Browser Automation 技能**

```bash
# 调用 Browser Automation 技能
/Skill Browser Automation "打开 {{账号链接}}，提取粉丝数、内容数、互动数据"
```

**方法C：WebFetch 简单抓取**

```python
WebFetch(url="{{账号链接}}", prompt="提取这个页面的：1. 粉丝数 2. 关注数 3. 内容数 4. 最近内容的点赞/评论数")
```

**方法D：手动输入（最后备选）**

如果所有自动化方法都失败：
1. 生成数据收集模板
2. 提示用户手动填写或跳过此平台

**数据抓取容错规则**：
- 粉丝数获取失败 → 标记为"未获取"，此平台不参与评分
- 内容数据获取失败 → 无法进行内容分析，跳过
- 部分数据获取成功 → 只分析成功获取的数据

### 步骤4：数据分析

基于成功获取的数据进行分析：

#### 4.1 计算核心指标

**基础指标**：
- 粉丝数：直接显示
- 粉丝增速：（当前粉丝 - 上期粉丝）/ 上期粉丝 × 100%
- 内容数：直接显示
- 发布频次：内容数 / 运营周数
- 互动率：（总点赞 + 总评论 + 总收藏）/ 粉丝数 × 100%
- 平均互动量：总互动量 / 内容数

**平台特化指标**：

| 平台 | 特化指标 | 计算公式 |
|------|---------|----------|
| 小红书 | 收藏率 | 收藏数 / 曝光量 × 100% |
| 小红书 | 评论率 | 评论数 / 曝光量 × 100% |
| 抖音 | 完播率 | 播放完成数 / 总播放数 × 100% |
| 抖音 | 涨粉率 | 新增粉丝 / 曝光量 × 100% |
| 视频号 | 转发率 | 转发数 / 曝光量 × 100% |
| 公众号 | 打开率 | 阅读数 / 粉丝数 × 100% |
| 公众号 | 分享率 | 分享数 / 阅读数 × 100% |

**行业基准对比**：

| 平台 | 互动率基准 | 粉丝增速基准 | 完播率基准 |
|------|------------|---------------|---------------|
| 小红书 | 3.5% | 2%/周 | - |
| 抖音 | 4.0% | 1.5%/周 | 45% |
| 视频号 | 2.0% | 1%/周 | - |
| 公众号 | 3.0% | 0.5%/周 | - |

#### 4.2 问题识别

将实际指标与基准对比，识别问题：

- 低于基准20%以上 → 🔴 严重问题
- 低于基准10-20% → ⚠️ 需要改进
- 高于基准 → ✅ 表现良好

**常见问题类型**：
1. 粉丝增长停滞 → 内容吸引力不足/发布频次低
2. 互动率低 → 内容质量差/受众不匹配
3. 完播率低（抖音）→ 开头不吸引人/内容节奏差
4. 打开率低（公众号）→ 标题封面不吸引人

#### 4.3 算法趋势分析

根据内置的算法趋势知识库，给出建议：

**小红书**（2026年5月）：
- 内容为王：减少硬广，增加真实分享
- 视频权重提升：增加视频内容比例至50%+
- 搜索流量增加：优化关键词布局

**抖音**（2026年5月）：
- 完播率为核心：前3秒抓眼球
- 同城流量扶持：添加定位标签
- 直播联动：固定直播时间

**视频号**（2026年5月）：
- 私域价值凸显：引导点赞分享
- 公众号联动：互相引流

**公众号**（2026年5月）：
- 打开率下降：优化标题和封面
- 算法推荐增强：保持更新频率

## 容错机制

| 失败环节 | 处理方式 | 是否中断 |
|---------|---------|----------|
| 账号搜索失败 | 标记为"未找到"，跳过此平台 | 否 |
| 数据抓取失败 | 标记为"未获取"，此平台数据留空 | 否 |
| 部分数据缺失 | 只分析成功获取的数据 | 否 |
| 历史数据缺失 | 不生成时间序列对比 | 否 |

**核心原则**：
- 任何环节失败都不中断整个诊断
- 只分析成功获取的数据
- 在报告中明确标注数据缺失情况

## 注意事项

1. **不要编造数据** - 如果数据获取失败，标记为"未获取"
2. **明确标注缺失** - 在报告中清楚说明哪些数据不可用
3. **基于现有数据给建议** - 不要基于未获取的数据给出建议
4. **尊重平台规则** - 不要尝试绕过登录或抓取私密数据
